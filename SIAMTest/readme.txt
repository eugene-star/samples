ЗАДАНИЕ
Необходимо спроектировать приложение, которое будет решать следующие задачи:
 1. Загружать файлы из разных источников - FTP, сетевой ресурс, веб-сервисы
 2. Приводить их к единому типу данных и сохранять их в БД
  
УСЛОВИЯ И ДОПУЩЕНИЯ
 1. Форматы файлов - csv, xml. Для веб-сервисов - типизированный возвращаемый результат. Содержание и структура всех получаемых данных произвольна.
 2. Конечный тип данных - класс должен включать в себя несколько типизированных свойств, которые также нужно сохранить в БД.
 3. Сохранение данных в БД можно эмулировать с помощью заглушек, структуру таблиц нужно просто привести в отдельном файле в виде диаграммы таблиц.
 4. Обращение к веб-сервисам можно эмулировать с помощью заглушек.
 5. Детальный код нужен для следующей функциональности:
 	a. Загрузка файла из источника
 	b. Чтение и преобразование записи (для файлов)
 6. Необходимо предусмотреть обработку всех возможных исключений и на псевдокоде привести решение проблемы. 

КОММЕНТАРИЙ ПО РЕАЛИЗАЦИИ
В строке ввода нужно ввести HTTP URL в случае веб-сервиса (произвольный, т.к. работает заглушка),
URL файла на FTP или путь к файлу на сетевом ресурсе в формате UNC.
По нажатию на кнопку файл загружается.
Тестовые примеры в папке FileSource, в формате XML и CSV.
